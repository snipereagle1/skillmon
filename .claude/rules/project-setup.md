# Project Setup & Development

## Environment Variables

Set in a `.env` file at the project root (never commit it).

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `EVE_CLIENT_ID` | Yes | — | OAuth client ID from EVE Developers portal |
| `EVE_CALLBACK_URL` | No | `http://localhost:1421/callback` | OAuth callback URL |

```env
EVE_CLIENT_ID=your_client_id_here
EVE_CALLBACK_URL=http://localhost:1421/callback
```

## Development Commands

```bash
pnpm tauri:dev        # Start full Tauri app with hot reload (runs typegen first)
pnpm tauri:build      # Production build (runs typegen first)
pnpm typegen          # Regenerate TypeScript bindings — run after any Rust struct change
pnpm typecheck        # tsc --noEmit
pnpm test             # vitest (TZ=UTC)
pnpm test:rust        # cargo test (in src-tauri/)
pnpm lint             # eslint
pnpm lint:rust        # cargo clippy -D warnings
pnpm format           # prettier + cargo fmt
pnpm verify           # typegen + (turbo) lint + lint:rust + typecheck + format checks + test + test:rust
```

Do not run a full build solely to regenerate frontend bindings — use `pnpm typegen`.

## ESI Client Generation

The ESI client (`src-tauri/src/esi/client.rs`, `types.rs`) is auto-generated from the EVE Online OpenAPI schema. Only regenerate when the ESI API schema changes.

```bash
./scripts/generate-esi.sh
```

Generated files are not checked into version control — they are regenerated at build time.

## TypeScript Bindings

Generated by `typeshare` via `pnpm typegen`. Output lands in `src/generated/`:

- `types.ts` — TypeScript type definitions mirroring Rust structs
- `index.ts` — re-exports from `types.ts`

Never edit these files manually. Use `pnpm typegen` to refresh.

## Database

- **Location**: SQLite in the Tauri app data directory (platform-specific)
  - Linux: `~/.local/share/com.skillmon/database.sqlite`
- **Mode**: WAL (Write-Ahead Logging) for better concurrency
- **Migrations**: `src-tauri/migrations/` — auto-applied on startup via `sqlx::migrate!()`

### Adding a Migration

1. Create `src-tauri/migrations/NNN_description.sql` (next sequential number)
2. Use `CREATE TABLE IF NOT EXISTS` for new tables
3. Use `ALTER TABLE` for schema changes
4. Commit the migration file — it runs automatically on next app start
