# Frontend Routing

TanStack Router with file-based routing. Routes are auto-discovered from `src/routes/`.

## Route Structure

```
src/routes/
├── __root.tsx                    # Root layout, auth events, global dialogs
├── index.tsx                     # /
├── overview.tsx                  # /overview
├── characters/
│   ├── index.tsx                 # /characters
│   ├── $characterId.tsx          # layout for /characters/$characterId
│   └── $characterId/
│       ├── index.tsx             # redirect → skill-queue
│       ├── skill-queue.tsx       # /characters/$characterId/skill-queue
│       ├── skills.tsx
│       ├── clones.tsx
│       ├── attributes.tsx
│       ├── plans.tsx
│       └── settings.tsx
└── plans/
    ├── index.tsx                 # /plans
    └── $planId.tsx               # /plans/$planId
```

## Route File Conventions

- `__root.tsx` — root layout (must export `createRootRoute`)
- `index.tsx` — index route for a directory
- `$param.tsx` — dynamic route parameter
- Never edit `routeTree.gen.ts` — auto-generated by the Vite plugin

## Creating a Route

```typescript
import { createFileRoute } from '@tanstack/react-router';

export const Route = createFileRoute('/my-route')({
  component: MyPage,
});
```

### With Parameters

```typescript
export const Route = createFileRoute('/characters/$characterId')({
  component: CharacterPage,
});

function CharacterPage() {
  const { characterId } = Route.useParams();
  // ...
}
```

### Layout Route (with Outlet)

```typescript
import { createFileRoute, Outlet } from '@tanstack/react-router';

export const Route = createFileRoute('/characters/$characterId')({
  component: CharacterLayout,
});

function CharacterLayout() {
  return (
    <div>
      <Outlet />
    </div>
  );
}
```

## Navigation

Always use `Link` — never `onClick` + `navigate` for simple links.

```typescript
import { Link } from '@tanstack/react-router';

<Link to="/characters/$characterId" params={{ characterId: String(id) }}>
  {({ isActive }) => <button className={isActive ? 'active' : ''}>...</button>}
</Link>
```

Programmatic navigation:

```typescript
const navigate = useNavigate();
navigate({ to: '/plans/$planId', params: { planId: String(id) } });
```

## Getting Route Parameters

```typescript
// Within route component
const { characterId } = Route.useParams();

// Outside route component (e.g. shared component)
import { useParams } from '@tanstack/react-router';
const params = useParams({ strict: false });
const characterId = params.characterId ? Number(params.characterId) : null;
```

## Layout Patterns

### Two-Column with Persistent Sidebar

```typescript
function CharactersIndexPage() {
  return (
    <div className="flex h-full gap-2 p-4">
      <div className="w-64 shrink-0 overflow-y-auto">
        <AccountSidebar />
      </div>
      <div className="flex-1 border rounded-lg overflow-hidden">
        <Outlet />
      </div>
    </div>
  );
}
```

### Tabs as Links (character detail views)

```typescript
<TabsList>
  <Link to="/characters/$characterId/skill-queue" params={{ characterId }}>
    {({ isActive }) => (
      <TabsTrigger value="skill-queue" data-state={isActive ? 'active' : 'inactive'}>
        Skill Queue
      </TabsTrigger>
    )}
  </Link>
</TabsList>
```

## Root Route

`__root.tsx` is responsible for:
- Global auth event listeners (`useAuthEvents`)
- Startup state handling
- Top-level navigation
- Global dialogs/drawers (`AddCharacterDialog`, `NotificationDrawer`, `SkillDetail`)
- Rendering `<Outlet />`
