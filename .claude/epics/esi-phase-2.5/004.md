# Task: Replace tauri-typegen with typeshare — annotate structs, delete commands.ts, update scripts

## Context

tauri-typegen generates `src/generated/types.ts` and `src/generated/commands.ts` from Tauri command signatures. It has a history of generating incorrect output (wrong serde rename handling, mismatched types). This task removes it entirely. typeshare replaces it for type generation; `invoke()` calls move into the hooks that use them; `commands.ts` is deleted.

This task runs after 001–003 because it needs the final Rust struct shapes to be stable before annotating everything.

## Work

### Rust — annotate all remaining command types

Audit every command file in `src-tauri/src/commands/` that will survive after task 007 (accounts, auth, settings, notifications, skill_plans, remaps, rate_limits, sde, overview, esi_snapshot). For every input/output struct and enum:

- Add `#[typeshare]` above `#[derive(...)]`.
- Add `use typeshare::typeshare;` to the file if not already present.
- Ensure `#[serde(rename_all = "camelCase")]` is present on all structs (typeshare reads this correctly; verify it matches what the frontend expects).

Files to audit (post-deletions from task 007 — surviving commands only):

- `commands/accounts.rs`
- `commands/auth.rs`
- `commands/settings.rs`
- `commands/notifications.rs`
- `commands/skill_plans.rs`
- `commands/remaps.rs`
- `commands/rate_limits.rs`
- `commands/sde.rs`
- `commands/characters.rs`
- `commands/esi_snapshot.rs` (new shape from task 003 — should already have annotations)

Do NOT annotate commands being deleted in task 007 (`skill_queues.rs`, `skills.rs`, `attributes.rs`, `clones.rs`, `location.rs`, `overview.rs`). Those files will be removed entirely.

### `Cargo.toml` and `tauri.conf.json`

- Remove `tauri-typegen` from `[build-dependencies]` or wherever it appears in `src-tauri/Cargo.toml`.
- Remove the `"tauri-typegen"` plugin block from `src-tauri/tauri.conf.json` (confirmed present at line 32).
- Verify `typeshare` crate is already present (it was added in Phase 1); if not, add it.

### `package.json` / `pnpm` scripts

- Replace `"typegen": "cargo tauri-typegen generate --output-path src/generated"` with `"typegen": "typeshare src-tauri/src --lang typescript --output-file src/generated/types.ts"`.
- Update `"typeshare"` script if it previously only targeted `events.ts` — now typeshare generates both `types.ts` and `events.ts`. Consider merging into a single `pnpm typegen` script or keeping separate outputs:
  - Option: `typeshare src-tauri/src --lang typescript --output-file src/generated/types.ts` for types only (events struct is already in `refresh/events.rs` and can share the same run).
  - Simplest: one typeshare run targeting `src-tauri/src` outputs all annotated types. Split into two output files only if typeshare supports it; otherwise merge into one `types.ts` and remove the separate `events.ts` (update all imports accordingly).
- Remove any `cargo tauri-typegen` references from `tauri:dev`, `tauri:build`, `verify` scripts.
- `CLAUDE.md` note: update the critical rules section to reflect the new typegen command.

### `src/generated/commands.ts`

Delete the file.

### `src/generated/types.ts`

After running the new typeshare command, verify:

- All types previously generated by tauri-typegen are present.
- All new types from tasks 001–003 (enriched event structs, `CharacterSnapshot`, etc.) are present.
- No types that were deleted (removed command return types) remain.

### `src/generated/events.ts`

If typeshare now generates a single unified output, this file merges into `types.ts`. Update all imports across the codebase from `@/generated/events` → `@/generated/types` (or whichever file contains the event types). If kept separate, ensure the typeshare run that generates it is consistent with the new script.

### Update CLAUDE.md

Update the "Never hand-edit `src/generated/`" rule to reflect:

- `commands.ts` no longer exists.
- `types.ts` generated by typeshare (not tauri-typegen).
- `events.ts` generated by typeshare (or merged into types.ts).

## Acceptance Criteria

- `cargo tauri-typegen` is not referenced anywhere in the project.
- `src/generated/commands.ts` does not exist.
- `pnpm typegen` (or equivalent) runs typeshare and produces correct `types.ts` (and optionally `events.ts`).
- All command input/output types for surviving commands are present in `src/generated/types.ts`.
- `pnpm typecheck` passes (no missing type imports).
- `pnpm lint:rust` passes.
