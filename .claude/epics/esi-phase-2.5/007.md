# Task: Remove deleted Rust commands + final audit

## Context

Tasks 005 and 006 remove all frontend callsites for the commands being deleted. This task removes the Rust implementations and performs a final audit to verify the data layer is clean end-to-end.

## Work

### Delete obsolete command handlers

Remove the following from `src-tauri/src/commands/`:

From `skill_queues.rs`: remove `get_skill_queue_for_character`, `get_skill_queues`, `get_training_characters_count`. Also remove `force_refresh_skill_queue` **only if** it has been superseded — it should be kept if still used as the manual refresh trigger.

From `skills.rs`: remove `get_character_skills_with_groups` (or the entire file if nothing else remains).

From `attributes.rs`: remove `get_character_attributes_breakdown` (or the entire file if nothing else remains).

From `clones.rs`: remove `get_clones` (or the entire file if nothing else remains).

From `location.rs`: remove `get_all_characters_locations` and `get_character_location`. The file likely goes away entirely.

From `overview.rs`: remove `get_training_characters_overview` — the entire file should be deleted.

### `src-tauri/src/commands/mod.rs`

Remove all registrations and `pub use` re-exports for deleted commands. The `tauri::generate_handler![]` macro call must be updated to exclude all removed commands.

### `src-tauri/src/lib.rs` or `main.rs`

Verify no deleted command names appear in the handler registration.

### Unused imports / dead code

Run `pnpm lint:rust` — fix any unused import warnings introduced by command deletions. Remove unused helper functions that were only called by deleted commands.

### Final audit checklist

**Backend:**

- [ ] `refresh/mod.rs` — every `Ok(None)` arm emits a re-hydration event (no silent no-ops).
- [ ] All surviving command types annotated with `#[typeshare]`.
- [ ] No tauri-typegen references in `Cargo.toml`, `build.rs`, `tauri.conf.json`, or any config file.
- [ ] `pnpm lint:rust` clean.
- [ ] `cargo build` succeeds.

**Frontend:**

- [ ] `src/generated/commands.ts` does not exist.
- [ ] No file imports from `@/generated/commands`.
- [ ] No `invoke()` call uses a command name that no longer exists in Rust.
- [ ] `esiEvents.ts` listeners are synchronous — no `async`, no `await`, no command calls.
- [ ] `__root.tsx` hydration calls `get_esi_snapshot` exactly once with no character ID argument.
- [ ] No `useMemo` in components performs ESI data joins across store slices.
- [ ] `useTrainingCharactersOverview` hook does not exist.
- [ ] `pnpm typecheck` passes.
- [ ] `pnpm lint` passes.
- [ ] `pnpm verify` passes end-to-end.

**Manual smoke test:**

- [ ] Launch app with warm SQLite cache + ESI network disabled → all screens show data, no spinners.
- [ ] Wait for a background refresh cycle to fire → UI updates without any user action.
- [ ] Force refresh (manual trigger) → data updates correctly.
- [ ] Add a new character → supervisor spawns refresher, events start flowing.
- [ ] Remove a character → supervisor cancels refresher, store clears that character's slices.

## Acceptance Criteria

- All items in the audit checklist above are checked.
- `pnpm verify` passes.
- App loads with data on launch (warm cache, no network).
- No permanent loading spinners under any normal operating condition.
