# Task: Enrich queue/skills/attributes supervisor payloads + fix Ok(None) starvation

## Context

The supervisor in `src-tauri/src/refresh/mod.rs` currently emits raw ESI shapes for queue, skills, and attributes. Listeners compensate by calling enrichment commands after every event. Additionally, `Ok(None)` arms (cache hit, no new ESI data) are silent no-ops — the store never receives data when the cache is warm, causing permanent loading spinners.

This task enriches the three simplest resource arms and fixes the starvation bug for all three.

## Work

### `src-tauri/src/refresh/events.rs`

Replace the raw payload structs for queue, skills, and attributes with enriched structs:

**Queue payload** (`QueuePayload`) — add per-item fields:

- `skill_name: Option<String>`
- `primary_attribute: Option<i64>`
- `secondary_attribute: Option<i64>`
- `rank: Option<i64>`
- `sp_per_minute: Option<f64>`
- `current_sp: Option<i64>`

Add top-level fields to `QueuePayload`:

- `character_name: String`
- `unallocated_sp: i64`
- `is_paused: bool`
- `is_omega: bool`
- `attributes: Option<AttributesData>` (reuse existing struct)

**Skills payload** (`SkillsPayload`) — add per-item fields to `SkillItem`:

- `skill_name: Option<String>`
- `group_id: Option<i64>`
- `group_name: Option<String>`
- `is_in_queue: bool`
- `is_injected: bool`

Add top-level to `SkillsPayload`:

- `character_name: String`
- `total_sp: i64`
- `unallocated_sp: Option<i64>`

**Attributes payload** (`AttributesPayload`) — replace flat `AttributesData` with a per-attribute breakdown struct:

- `AttributeBreakdown { base, implants, remap, accelerator, total }` (mirrors `CharacterAttributesBreakdown` from existing types)
- `bonus_remaps: Option<i32>`
- `last_remap_date: Option<String>`
- `accrued_remap_cooldown_date: Option<String>`
- `character_name: String`

Annotate all new/changed structs with `#[typeshare]`.

### `src-tauri/src/refresh/mod.rs`

**Queue arm**:

- After fetching `queue_data`, look up skill names, ranks, and attribute IDs via SDE queries (reuse `utils::get_type_names` and `utils::get_skill_attributes`).
- Compute `sp_per_minute` using character attributes from DB (`db::get_character_attributes`) and `is_omega` from `db::get_character`.
- Compute `current_sp` using the same logic as `skill_queues.rs::build_character_skill_queue`.
- Compute `is_paused` from queue items.
- `Ok(None)` arm: read last-known queue from SQLite cache (`cache::get_cached_response` → deserialize), enrich with same logic, emit. If no cached row exists, skip.

**Skills arm**:

- After fetching `skills_data`, look up skill names and group info via SDE queries.
- Join with current queue slice to compute `is_in_queue` per skill.
- `is_injected` is always true for skills returned by the ESI skills endpoint.
- `Ok(None)` arm: read `db::get_character_skills`, enrich, emit.

**Attributes arm**:

- After fetching `attrs`, read implant data from DB (`db::get_clone_implants` for active clone) to compute implant bonuses per attribute.
- Read planned remaps from DB to compute remap column.
- Read booster/accelerator data if tracked (or default to 0).
- Compute `total` per attribute.
- `Ok(None)` arm: read `db::get_character_attributes`, enrich, emit.

### Extract enrichment helpers

Create `src-tauri/src/refresh/enrichment.rs` (or add to an existing shared module) with functions:

- `enrich_queue(pool, character_id, raw_queue) -> EnrichedQueue`
- `enrich_skills(pool, character_id, raw_skills, queue_skill_ids) -> EnrichedSkills`
- `enrich_attributes(pool, character_id, raw_attrs) -> EnrichedAttributes`

These functions are reusable from the snapshot command (task 003).

## Acceptance Criteria

- `character:{}:queue` event payload includes `skill_name`, `sp_per_minute`, `character_name`, `is_paused`, `is_omega`.
- `character:{}:skills` event payload includes `skill_name`, `group_name`, `is_in_queue` per skill.
- `character:{}:attributes` event payload includes per-attribute `{ base, implants, remap, accelerator, total }`.
- When cache is warm (no ESI fetch), all three arms still emit an event with the last-known DB row enriched.
- `pnpm typeshare` regenerates `events.ts` without errors.
- `pnpm lint:rust` passes.
