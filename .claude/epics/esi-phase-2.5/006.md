# Task: Update hooks — inline invoke calls, remove dead hooks, fix useOverview pattern

## Context

With `commands.ts` deleted and ESI live-data hooks now reading from the Zustand store (populated by events and snapshot), several hooks need to be updated or removed. Hooks for deleted commands become dead code. Remaining command hooks need their `invoke()` calls moved inline. The `useOverview` re-fetch-on-queue-change pattern is eliminated — overview data now comes directly from the queue slice.

## Work

### Remove dead hooks

Delete hooks that wrapped deleted ESI commands and are now fully replaced by store reads:

- `src/hooks/tauri/useCharacterSkills.ts` — was a store read, still fine; update type to match new `SkillsPayload` shape.
- `src/hooks/tauri/useAttributes.ts` — still a store read; update type.
- `src/hooks/tauri/useClones.ts` — still a store read; update type.
- `src/hooks/tauri/useLocationsOverview.ts` — still a store read; update type.
- `src/hooks/tauri/useSkillQueue.ts` — still a store read; update type.
- `src/hooks/tauri/useOverview.ts` — **delete this file**. The `getTrainingCharactersOverview` command is gone. Components that used `useTrainingCharactersOverview` now read directly from the queue store slice.
- `src/hooks/tauri/useForceRefreshSkillQueue.ts` — **keep**; update to use inline `invoke('force_refresh_skill_queue', { characterId })` instead of the deleted command wrapper.

### Update store-reading hooks

For hooks that read from the Zustand store, update the return type to match the new enriched payload types (now from `@/generated/types`):

- `useSkillQueue` → returns `QueuePayload | null`
- `useCharacterSkills` → returns `SkillsPayload | null`
- `useAttributes` → returns `AttributesPayload | null` (AttributeBreakdown per attribute is now in the payload)
- `useClones` → returns `ClonePayload | null`
- `useLocationsOverview` → returns `LocationPayload[]`

### Move inline invoke calls

For all surviving command hooks that previously imported from `commands.ts`:

- Replace `import { commandName } from '@/generated/commands'` with `import { invoke } from '@tauri-apps/api/core'`.
- Replace `commandName(params)` with `invoke<ReturnType>('command_name', params)`.
- Ensure the return type is imported from `@/generated/types`.

Hooks that need this treatment:

- `useAccountsAndCharacters.ts` — `getAccountsAndCharacters` → inline invoke.
- `useNotifications.ts` — `getNotifications` → inline invoke.
- `useNotificationSettings.ts` — `getNotificationSettings`, `upsertNotificationSetting` → inline invoke.
- `useDismissNotification.ts` — `dismissNotification` → inline invoke.
- `useRemaps.ts` — `getCharacterRemaps`, `getPlanRemaps`, `saveRemap`, `deleteRemap` → inline invoke. Also remove the `getCharacterAttributesBreakdown` re-fetch in `useSaveRemap`/`useDeleteRemap` mutation callbacks — the attributes event will fire after the supervisor's next cycle picks up the change. If immediate reflection is needed, the mutation callback can invoke `get_esi_snapshot` for just that character (or accept the short delay).
- `useSkillPlans.ts` — inline all plan command invocations.
- `useSettings.ts` — inline settings commands.
- `useSdeSkills.ts` — inline SDE commands.
- `useAuthEvents.ts` — inline auth commands; remove the `getCharacterAttributesBreakdown` re-fetch (same reasoning as remaps).
- `useLogoutCharacter.ts` — inline.
- `useStartEveLogin.ts` — inline.
- `useStartupState.ts` — inline.
- `useOptimization.ts`, `usePlanComparison.ts`, `usePlanComparisonAll.ts`, `useSimulation.ts`, `useSkillDetails.ts` — inline all plan/simulation commands.

### Fix direct component imports from commands.ts

Several components import directly from `@/generated/commands` rather than going through hooks. These must all be updated before `commands.ts` can be deleted:

- **`src/components/Clones/index.tsx`** — imports `getTypeNames`. Move `invoke<TypeNameEntry[]>('get_type_names', { typeIds })` inline or extract a `useTypeNames` hook if used in multiple places.
- **`src/components/SkillPlans/SimulationTimeline.tsx`** — imports `getTypeNames`. Same treatment as above.
- **`src/components/SkillPlans/PlanEditor.tsx`** — imports `exportSkillPlanText`, `exportSkillPlanXml`. Move inline into the component's export handler callbacks.
- **`src/components/LocationDemo.tsx`** — imports `getCharacterLocation`. This command is deleted; assess whether `LocationDemo` is a live component or a leftover. If live, replace with the location store slice; if dead code, delete the file.

### Fix OverviewTable / AccountsWithoutTraining

`OverviewTable` used `useTrainingCharactersOverview` (now deleted). Update to read directly from the queue store:

- `useEsiStore(state => state.queues)` → derive training characters from queue slices.
- Fields previously returned by `TrainingCharacterOverview` (`account_name`, `current_skill_name`, `sp_per_hour`, `has_implants`, `has_booster`) are now in the `QueuePayload` from task 001. Update `OverviewTableRow` to use the new field names.

### Fix SimulationTab

`SimulationTab` uses `useAttributes` to seed the simulation profile's initial implant/remap values. With the new `AttributesPayload` shape (breakdown per attribute), update the seeding logic:

- `characterAttributes.charisma.implants` → same field path if breakdown struct is preserved, or adjust to new shape.
- Verify the simulation still seeds correctly after type changes.

### Fix AccountSidebar

Remove the `useMemo` that joined `allCharacters` with `queues`. The queue slice now carries `character_name` and `is_paused` directly. The sidebar can read `useEsiStore(state => state.queues)` and index by character ID without needing the `allCharacters` list for the join.

## Acceptance Criteria

- No file in `src/hooks/tauri/` imports from `@/generated/commands` (file is deleted).
- `useTrainingCharactersOverview` and `useOverview.ts` do not exist.
- All surviving hooks import `invoke` from `@tauri-apps/api/core` and types from `@/generated/types`.
- `OverviewTable` derives training characters from the queue store without a command re-fetch.
- `AccountSidebar` has no cross-slice `useMemo` join.
- `SimulationTab` seeds correctly from the new attributes payload shape.
- `pnpm typecheck` passes.
- `pnpm lint` passes.
