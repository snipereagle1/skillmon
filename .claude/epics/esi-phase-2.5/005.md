# Task: Rewrite esiEvents.ts as pure setters + rewrite mount hydration in \_\_root.tsx

## Context

`src/lib/esiEvents.ts` ignores payloads, calls enrichment commands after events. Supervisor now delivers enriched payloads (tasks 001, 002) — listeners become pure setters. No async, no network.

`src/routes/__root.tsx` hydration fan-outs 5+ commands per character. New `get_esi_snapshot` (task 003) replaces with single `invoke()` returning all characters.

## Work

### `src/lib/esiEvents.ts`

Rewrite every listener to pure setter. Receives typed payload from `@/generated/types` or `@/generated/events`, calls matching store setter directly.

Remove deleted command imports. Remove `/* eslint-disable no-await-in-loop */` (no async in listeners).

New shape:

```typescript
const unlistenQueue = await listen<QueuePayload>(
  `character:${characterId}:queue`,
  ({ payload }) => useEsiStore.getState().setQueue(characterId, payload)
);

const unlistenSkills = await listen<SkillsPayload>(
  `character:${characterId}:skills`,
  ({ payload }) => useEsiStore.getState().setSkills(characterId, payload)
);

// ... same pattern for attributes, location, clones
```

`notifications:new` listener unchanged (invalidates TanStack Query, not ESI store).

Update `useEsiStore` setter types (`setQueue`, `setSkills`, etc.) to accept enriched payload types from `@/generated/types`. Store slices now hold enriched event payload shapes directly.

### `src/stores/esiStore.ts`

Update type params per store slice to match enriched payload types:

- `queues: Record<number, ResourceSlice<QueuePayload>>`
- `skills: Record<number, ResourceSlice<SkillsPayload>>`
- `attributes: Record<number, ResourceSlice<AttributesPayload>>`
- `locations: Record<number, ResourceSlice<LocationPayload>>`
- `clones: Record<number, ResourceSlice<ClonePayload>>`

Remaps slice unchanged (command-driven).

### `src/routes/__root.tsx`

Replace per-character hydration fan-out with single `invoke<CharacterSnapshot[]>('get_esi_snapshot')`:

```typescript
const snapshots = await invoke<CharacterSnapshot[]>('get_esi_snapshot');
for (const snapshot of snapshots) {
  const id = snapshot.character_id;
  if (snapshot.queue) store.setQueue(id, snapshot.queue);
  if (snapshot.skills) store.setSkills(id, snapshot.skills);
  if (snapshot.attributes) store.setAttributes(id, snapshot.attributes);
  if (snapshot.clones.length > 0) store.setClones(id, snapshot.clones);
  if (snapshot.location) store.setLocation(id, snapshot.location);
  if (snapshot.remaps.length > 0) store.setRemaps(id, snapshot.remaps);
}
```

Per-slice error handling: if `invoke` throws, set `lastError` on all slices for all characters — flips `isLoading` false, renders error state not permanent spinner.

`getAccountsAndCharacters` **kept** as prerequisite for character IDs to pass to `bootstrapEsiEvents`. Not used to drive hydration loop. New sequence:

```typescript
const accountsData = await getAccountsAndCharacters(); // lightweight — needed for character IDs
const characterIds = [
  ...accountsData.unassigned_characters,
  ...accountsData.accounts.flatMap((a) => a.characters),
].map((c) => c.character_id);

try {
  const snapshots = await invoke<CharacterSnapshot[]>('get_esi_snapshot');
  // populate store from snapshots...
} catch (err) {
  // set lastError on all known character slices so isLoading flips false
  for (const id of characterIds) {
    setError('queues', id, String(err));
    setError('skills', id, String(err));
    // etc.
  }
}

cleanup = await bootstrapEsiEvents(queryClient, characterIds);
```

Guarantees listeners registered for all known characters regardless of snapshot success/failure.

Remove all deleted command imports from `__root.tsx`.

`bootstrapEsiEvents` calling convention unchanged.

## Acceptance Criteria

- `esiEvents.ts`: zero `invoke()` calls, zero `async` listener callbacks.
- All listener payloads typed via generated event payload types.
- Mount hydration calls `invoke('get_esi_snapshot')` once. No per-character loop.
- `get_esi_snapshot` failure → all store slices for all characters have `lastError` set. No permanent spinners.
- `pnpm typecheck` passes.
- `pnpm lint` passes. No unused imports from deleted commands.
