# Task: Migrate useOverview (Training Characters) Hook to Zustand

## Description

Migrate `useTrainingCharactersOverview` from React Query (5-minute `refetchInterval`) to Zustand. This hook aggregates queue + skills data per character into a `TrainingCharacterOverview[]` summary. Because queue and skills slices are both in the store by this point, the hook can derive its output by composing store selectors — no separate event or store slice needed.

## Acceptance Criteria

- [ ] `useTrainingCharactersOverview()` reads from queue and skills slices in `useEsiStore` — no `useQuery`, no `refetchInterval`
- [ ] Output shape `TrainingCharacterOverview[]` is preserved and compatible with existing consumers
- [ ] If `getTrainingCharactersOverview` (the Tauri command) is purely a computed aggregation of queue+skills, remove the command call entirely and derive in the hook; if it has additional logic not in the store, retain as a one-shot (non-live) call and document why
- [ ] Phase 1 RQ adapter for overview (if any) removed
- [ ] Consumer components unchanged
- [ ] `pnpm typecheck` passes, `pnpm test` passes

## Technical Details

Current implementation calls `getTrainingCharactersOverview()` with `refetchInterval: 5 * 60 * 1000`. Check what this command returns vs. what's already in the queue/skills slices. If `TrainingCharacterOverview` is a projection of skill queue + trained skills, derive it in the hook from `useEsiStore`. If the command does server-side join logic not replicated in the store, keep the command call but remove the interval (call once on mount).

Check Rust command implementation in `src-tauri/src/commands/overview.rs` to understand what it computes.

Verify call sites: `grep -r "useTrainingCharactersOverview\|useOverview" src/`.
