# Task: Migrate Skill Queue Hook to Zustand

## Description

Migrate `useSkillQueue` from React Query to the Zustand ESI store. Update the Phase 1 event listener for `character:{id}:queue` in `src/lib/esiEvents.ts` to call the store setter instead of `queryClient.setQueryData`. Remove the RQ adapter for queue entirely.

This is the first hook migration and the most impactful (queue data drives notifications). Must be sequential with other esiEvents.ts migrations to prevent dual-system violations.

## Acceptance Criteria

- [ ] `useSkillQueue(characterId)` reads from `useEsiStore(state => state.queues[characterId]?.data ?? null)` — no `useQuery` call
- [ ] Hook return shape is compatible with prior RQ version: `{ data, isLoading, error }` equivalent
- [ ] `esiEvents.ts` queue listener calls `useEsiStore.getState().setQueue(...)` — no `queryClient.setQueryData` for queue
- [ ] Phase 1 RQ adapter for queue removed (no `queryKeys.skillQueue` usage in event bootstrap)
- [ ] `refetchInterval` option removed from `useSkillQueue` signature (no longer meaningful)
- [ ] Consumer components unchanged (verified by grep — no call sites modified)
- [ ] `pnpm typecheck` passes, `pnpm test` passes

## Technical Details

Current `useSkillQueue` signature:

```ts
export function useSkillQueue(
  characterId: number | null,
  options?: { refetchInterval?: number | false }
);
```

New signature:

```ts
export function useSkillQueue(characterId: number | null);
```

Return value must still expose `data`, `isLoading` (derive from `data === null && no error`), and `error` for call-site compatibility.

Check all usages: `grep -r "useSkillQueue" src/` before and after to confirm no call site breaks.
