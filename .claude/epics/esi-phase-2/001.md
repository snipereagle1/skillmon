# Task: Create Zustand ESI Store + Wire Initial Hydration

## Description

Create `src/stores/esiStore.ts` with per-resource slices keyed by `character_id`. Wire the initial hydration call so the store is populated from SQLite (`get_esi_snapshot` command from Phase 1) before event listeners take over â€” ensuring the UI is never blank on launch.

This is the prerequisite for all hook migrations. No hooks are changed in this task.

## Acceptance Criteria

- [ ] `src/stores/esiStore.ts` exists with slices: `queues`, `skills`, `locations`, `attributes`, `clones`, `remaps`
- [ ] Each slice type: `Record<number, { data: T | null, lastUpdatedAt: string | null, lastError: string | null }>`
- [ ] Typed setter per resource: `setQueue(characterId, data)`, `setSkills(...)`, etc.
- [ ] `clearCharacter(characterId)` action removes all slices for a removed character

## Technical Details

Store shape:

```ts
useEsiStore = {
  queues:    Record<CharacterId, { data: CharacterSkillQueue | null, lastUpdatedAt: string | null, lastError: string | null }>,
  skills:    Record<CharacterId, { data: CharacterSkillsResponse | null, lastUpdatedAt: string | null, lastError: string | null }>,
  locations: Record<CharacterId, { data: CharacterLocationOverview | null, lastUpdatedAt: string | null, lastError: string | null }>,
  attributes: Record<CharacterId, { data: CharacterAttributesBreakdown | null, ... }>,
  clones:    Record<CharacterId, { data: CloneResponse[] | null, ... }>,
  remaps:    Record<CharacterId, { data: Remap[] | null, ... }>,
  setQueue(characterId: number, data: CharacterSkillQueue): void,
  setSkills(characterId: number, data: CharacterSkillsResponse): void,
  setLocation(characterId: number, data: CharacterLocationOverview): void,
  setAttributes(characterId: number, data: CharacterAttributesBreakdown): void,
  setClones(characterId: number, data: CloneResponse[]): void,
  setRemaps(characterId: number, data: Remap[]): void,
  setError(resource: ResourceKey, characterId: number, error: string): void,
  clearCharacter(characterId: number): void,
}
```

Types come from `src/generated/types`. `get_esi_snapshot` command returns a snapshot object â€” check Phase 1 implementation for exact shape.

Setters must write `lastUpdatedAt: new Date().toISOString()` on success and `lastError` on failure.
