# Task: Phase 2 Audit — Verify No Live-Data Leaks into React Query

## Description

Final verification pass confirming the migration is complete and no invariants are violated. Audit the codebase for any remaining `refetchInterval` on live-data queries, any `queryClient.setQueryData` for ESI-derived data, and any RQ query for resources now owned by the Zustand store. Remove dead query keys. Confirm the dual-system invariant (no resource has both a Zustand listener and an RQ listener writing to it).

## Acceptance Criteria

- [ ] No `refetchInterval` remains in any live-data hook (`useSkillQueue`, `useCharacterSkills`, `useLocationsOverview`, `useAttributes`, `useClones`, `useCharacterRemaps`, `useOverview`)
- [ ] No `queryClient.setQueryData` for ESI live data in `esiEvents.ts` or elsewhere
- [ ] `queryKeys.ts` has no dead keys for migrated resources (remove them or document if still used by something legitimate)
- [ ] All `useQuery` calls in `hooks/tauri/` are for mutations, SDE/static reads, settings, or one-shot fetches — not live ESI data
- [ ] `esiEvents.ts` contains only store setter calls; no `queryClient` import if nothing else needs it
- [ ] `pnpm verify` passes clean (typegen + lint:full + format:check + typecheck)
- [ ] Manual smoke test: launch app minimized, confirm queue/skills/location update in store when refresh fires (check DevTools or add temporary log)

## Technical Details

Audit commands to run:

```bash
grep -r "refetchInterval" src/hooks/tauri/
grep -r "setQueryData" src/
grep -r "queryKeys\." src/lib/esiEvents.ts
grep -rn "useQuery" src/hooks/tauri/ | grep -v "Mutation\|useMutation\|#"
```

Expected results: all empty (or only legitimate non-live hits).

Dead query key check: for each key removed from `queryKeys.ts`, grep for usages — if nothing references it, delete it.
