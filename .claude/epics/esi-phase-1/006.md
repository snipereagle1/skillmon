# Task: Emit Tauri events per resource after each fetch

## Description

After each successful ESI fetch in the `RefresherTask` loop, emit a Tauri event for each resource with the full resource value as payload. Payloads are typed against the structs defined in Task 002 and generated into `src/generated/events.ts`. Event names follow `character:{id}:{resource}` convention.

## Acceptance Criteria

- [ ] After a successful fetch cycle, one event emitted per resource: `character:{id}:queue`, `character:{id}:skills`, `character:{id}:location`, `character:{id}:public`, `character:{id}:attributes`, `character:{id}:clones`, `character:{id}:remaps`
- [ ] Each event payload is the corresponding typeshare struct from Task 002 (includes `character_id` + resource data)
- [ ] Events only emitted when data was successfully fetched (not on error/cache-hit-no-change)
- [ ] `app.emit(event_name, &payload)` errors are logged but do not abort the loop
- [ ] Frontend can receive and deserialize these events using types from `src/generated/events.ts`
- [ ] Event emission verified with a temporary `console.log` in the frontend listener (removed before merge)

## Technical Details

Event name construction:

```rust
let event_name = format!("character:{}:queue", character_id);
app.emit(&event_name, &QueuePayload { character_id, queue: fetched_queue })?;
```

Payload structs come from the `refresh::events` module created in Task 002.

## Dependencies

Task 002 (payload structs + generated events.ts). Task 004 (refresher loop fetches data).
