# Task: Wire supervisor into Tauri app lifecycle

## Description

Integrate `RefreshSupervisor` into the Tauri app setup in `src-tauri/src/lib.rs`. Spawn the supervisor on app start, seed it with all existing characters from the DB, register it as Tauri state, wire the existing `force_refresh` command to call `supervisor.poke(character_id)`, and register a `RunEvent::ExitRequested` / app close hook that cancels all refreshers before exit.

## Acceptance Criteria

- [ ] `RefreshSupervisor` created and registered as Tauri managed state in `lib.rs` setup
- [ ] On app start: `db::get_all_characters` called; supervisor spawns one refresher per character
- [ ] Existing `force_refresh_skill_queue` command (or equivalent) updated to call `supervisor.poke(character_id)` instead of / in addition to its current behavior
- [ ] `app.on_window_event` or `RunEvent::Exit` handler calls `supervisor.cancel_all()` on app close
- [ ] Character add flow (SSO callback completion) triggers `supervisor.spawn_character(id)` for the new character
- [ ] Character remove flow triggers `supervisor.cancel_character(id)`
- [ ] `cargo check` passes; app starts and supervisor logs first refresh tick for each character

## Technical Details

In `lib.rs` `setup` closure:

```rust
let supervisor = RefreshSupervisor::new();
// seed with existing characters
for char_id in db::get_character_ids(&pool).await? {
    supervisor.spawn_character(char_id, pool.clone(), app_handle.clone(), rate_limits.clone());
}
app.manage(supervisor);
```

Shutdown in `RunEvent::ExitRequested`:

```rust
RunEvent::ExitRequested { .. } => {
    let supervisor = app.state::<RefreshSupervisor>();
    supervisor.cancel_all();
}
```
