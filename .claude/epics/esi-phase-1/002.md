# Task: Define event payload Rust structs with typeshare

## Description

Create the Rust structs for all Tauri event payloads used by the Phase 1 event bridge. Annotate them with `#[typeshare]` and `#[derive(Serialize, Deserialize)]`. Run `pnpm typeshare` to generate `src/generated/events.ts`. This establishes the type contract the frontend event listeners will import from.

Event naming: `character:{id}:{resource}` and `notifications:new`.

## Acceptance Criteria

- [ ] New file `src-tauri/src/refresh/events.rs` (or `src-tauri/src/events.rs`) contains payload structs for: `queue`, `skills`, `location`, `public`, `attributes`, `clones`, `remaps`, and `notifications:new`
- [ ] All structs have `#[typeshare]`, `#[derive(Clone, Serialize, Deserialize)]`
- [ ] `#[serde(rename_all = "camelCase")]` applied where appropriate so TS output matches frontend conventions
- [ ] `pnpm typeshare` generates `src/generated/events.ts` with correct TypeScript types for all structs
- [ ] Generated `events.ts` is committed (never hand-edited)
- [ ] Struct field types match the shapes returned by existing `esi_helpers` functions

## Technical Details

Payload structs live in the refresh module. Each struct wraps the ESI data type already used by `esi_helpers`. Example:

```rust
#[typeshare]
#[derive(Clone, Serialize, Deserialize)]
pub struct QueuePayload {
    pub character_id: i64,
    pub queue: Vec<SkillQueueItem>,  // reuse existing ESI type if typeshare-compatible
}
```

If existing ESI types (from `esi/types.rs`) are too complex for typeshare, define simplified payload structs that mirror the shape the frontend already consumes.
