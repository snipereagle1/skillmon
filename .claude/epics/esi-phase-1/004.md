# Task: Implement RefresherTask loop with ESI fetches and persist

## Description

Replace the Task 003 stub loop body with a full per-character refresh cycle: call `ensure_valid_access_token`, fetch all live ESI endpoints via `esi_helpers`, diff against current DB state, persist changes, then sleep until `min(expires) + jitter` (clamped 30s floor, 1h ceiling). On ESI error (5xx / network), log and backoff 5 minutes before retrying.

## Acceptance Criteria

- [ ] Loop calls `auth::ensure_valid_access_token(pool, character_id)` at the top of each cycle
- [ ] Fetches: skill queue, character skills, location, public info, attributes, clones, remaps — via existing `esi_helpers` functions
- [ ] Results persisted to SQLite via existing `db` layer
- [ ] Sleep computed from `expires` header values returned by `esi_helpers`; clamped to 30s floor and 1h ceiling
- [ ] Jitter applied to sleep (e.g., ±10% random variance) to avoid synchronized thundering herd
- [ ] On ESI error: logs the error with `eprintln!` or `tracing`, sleeps 5 minutes, then continues loop (does not panic or exit)
- [ ] Cancellation token checked in `select!` so the loop exits cleanly when cancelled
- [ ] Poke via `Arc<Notify>` wakes the loop early (skips the sleep)
- [ ] `cargo check` passes; existing tests still pass

## Technical Details

Sleep logic:

```rust
let sleep_duration = expires_durations
    .into_iter()
    .min()
    .unwrap_or(Duration::from_secs(300))
    .max(Duration::from_secs(30))
    .min(Duration::from_secs(3600));

tokio::select! {
    _ = tokio::time::sleep(sleep_duration) => {}
    _ = notify.notified() => {}
    _ = cancel.cancelled() => { return; }
}
```
