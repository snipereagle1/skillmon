# Task: Add tokio-util + typeshare deps and tooling setup

## Description

Add `tokio-util` (CancellationToken) and `typeshare` (event payload type generation) as Rust dependencies. Set up `typeshare-cli` in the dev workflow, create `typeshare.toml` config, add `pnpm typeshare` script to `package.json`, and update `pnpm verify` to run typeshare. This is the prerequisite for all other Phase 1 tasks.

## Acceptance Criteria

- [ ] `tokio-util` with `sync` feature added to `src-tauri/Cargo.toml`
- [ ] `typeshare` crate added to `src-tauri/Cargo.toml`
- [ ] `typeshare.toml` created at project root with output configured to `src/generated/events.ts`
- [ ] `pnpm typeshare` script added to `package.json` (runs `typeshare-cli`)
- [ ] `pnpm verify` updated to include `pnpm typeshare` before typecheck
- [ ] CLAUDE.md Critical Rules updated: note that `events.ts` is generated by typeshare (`pnpm typeshare`), `commands.ts`/`types.ts` by tauri-typegen (`pnpm typegen`)
- [ ] `pnpm typeshare` runs without error (empty output is fine before structs are annotated)

## Technical Details

`typeshare.toml`:

```toml
[typescript]
output_file = "src/generated/events.ts"
```

`package.json` script:

```json
"typeshare": "typeshare-cli src-tauri/src --lang typescript --output-file src/generated/events.ts"
```

`tokio-util` needs at minimum the `sync` feature for `CancellationToken`. Full features also acceptable given `tokio = { features = ["full"] }` already present.
