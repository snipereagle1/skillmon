# Task: Remove refetchInterval and integration verification

## Description

Remove `refetchInterval` from all live-data query hooks now that Rust drives the refresh cadence. Then do end-to-end integration verification: confirm live data updates arrive via events, and crucially confirm notifications fire on schedule with the app window minimized or on a different desktop.

## Acceptance Criteria

- [ ] `refetchInterval` removed from `useOverview.ts`, `useLocationsOverview.ts`, `useSkillQueue.ts`
- [ ] No `refetchInterval` remains in `src/hooks/tauri/` for ESI live-data hooks (grep check)
- [ ] `pnpm typecheck` passes after removals
- [ ] `pnpm lint` passes
- [ ] Manual test: app running, window minimized for 10+ minutes — low-queue notification fires as expected
- [ ] Manual test: live data (skill queue countdown) visibly updates when window is brought to foreground after being minimized
- [ ] Manual test: force-refresh (via UI button if present) pokes the refresher and data updates immediately
- [ ] No regressions in unrelated hooks (mutation hooks, SDE hooks still function normally)

## Technical Details

Hooks to modify:

- `src/hooks/tauri/useOverview.ts` — remove `refetchInterval: 5 * 60 * 1000`
- `src/hooks/tauri/useLocationsOverview.ts` — remove `refetchInterval: 60_000`
- `src/hooks/tauri/useSkillQueue.ts` — remove `refetchInterval` option plumbing (callers no longer pass it)

After removal, these hooks rely solely on `queryClient.setQueryData` calls from `esiEvents.ts` (Task 009) to receive updates.

Verification grep:

```bash
grep -rn "refetchInterval" src/hooks/tauri/
```

Should return zero hits for live-data hooks.
