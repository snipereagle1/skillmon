# Task: Add get_esi_snapshot hydration command

## Description

Add a new Tauri command `get_esi_snapshot(character_id: i64)` that reads the current SQLite state for all live resources (queue, skills, location, public info, attributes, clones, remaps) and returns them in a single response. This is called by the frontend on app mount to hydrate the React Query cache (Phase 1) or Zustand store (Phase 2) before the first ESI push arrives, so the UI is never blank on startup.

## Acceptance Criteria

- [ ] New command `get_esi_snapshot` in `src-tauri/src/commands/` returns all live resource data for a character from SQLite
- [ ] Return type is a struct containing all resource fields (nullable — some may not be populated on first launch)
- [ ] Command registered in `src-tauri/src/commands/mod.rs` and exposed via `tauri::generate_handler!`
- [ ] Return struct annotated with `#[typeshare]` and included in `pnpm typeshare` output in `events.ts` (or a separate generated file if cleaner)
- [ ] Frontend: `pnpm typegen` regenerated to include the new command wrapper in `commands.ts`
- [ ] Frontend: command called in `src/main.tsx` or app bootstrap; result used to seed `queryClient` cache for all live-data query keys
- [ ] No ESI network calls made — reads only from SQLite

## Technical Details

```rust
#[tauri::command]
pub async fn get_esi_snapshot(
    character_id: i64,
    pool: tauri::State<'_, db::Pool>,
) -> Result<EsiSnapshot, String> {
    Ok(EsiSnapshot {
        character_id,
        queue: db::get_skill_queue(&pool, character_id).await.ok().flatten(),
        skills: db::get_character_skills(&pool, character_id).await.ok().flatten(),
        location: db::get_location(&pool, character_id).await.ok().flatten(),
        // ... etc
    })
}
```
